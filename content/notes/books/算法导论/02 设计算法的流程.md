设计、证明、分析是处理计算问题的基本步骤。本章以排序问题为例，遍历了这三个基本步骤。首先给出了一个算法（插入排序），证明了它的正确性，分析了它的运行时间。然后使用另一种设计思想（分治）设计了另一种算法（归并排序），同样证明了它的正确性，分析了它的运行时间。

## 2.1 插入排序

前文提到，排序问题的定义是：

> **输入**：含有 $n$ 个数字的序列 $\lang a_1, a_2, \dots, a_n \rang$。
>
> **输出**：输入序列的某个满足 $a'_1 \leqslant a'_2 \leqslant \dots \leqslant a'_n$ 的排列 $\lang a'_1, a'_2, \dots, a'_n \rang$。

插入排序是为少量元素排序的高效算法。该算法与抓牌类似，手中的牌保持有序，每次从牌堆中抓取一张，插入到手牌中的适当位置使其仍然有序。**伪代码**（pseudocode）如下：

``` plaintext
INSERTION-SORT(A)
  for j = 2 to A.length
    // 备份 A[j]，因为后续步骤中 A[j] 可能会被覆盖掉
    key = A[j]

    i = j - 1
    while i > 0 and A[i] > key
      // 如果 A[i] 比 A[j] 大，那么将 A[i] 往后挪动一格
      A[i + 1] = A[i]
      i = i - 1
    
    // 把 A[j] 插入到此处
    A[i + 1] = key
```

### 循环不变和插入排序的正确性

`A[i..j - 1]` 的以下性质是**循环不变**（loop invariant）的：在算法中 `for` 循环每次迭代的开始，`A[1..j - 1]` 中包含原数组中 `A[1..j - 1]` 中的所有元素，并且是有序的。

循环不变性可以用于证明为什么插入排序是正确的。类似于数学归纳法，只需要证明以下三点：

- **初始**（initialization）：在第一次迭代开始之前，循环不变成立。
- **维护**（maintenance）：如果本次迭代开始之前循环不变成立，那么下一次迭代开始之前循环不变也成立。
- **终止**（termination）：当算法终止时，循环不变性能够用于证明算法正确。

根据以上方法，可以借助循环不变性证明 `INSERTION-SORT` 算法是正确的。详细的证明过程略。

## 2.2 算法分析

算法**分析**（analyze）用于预测算法需要的资源。

### 插入排序的分析

**输入规模**（input size）的具体表示取决于要研究的问题。在排序问题中，输入规模一般指的是需要排序的元素数量。算法的**运行时间**（running time）指的是在特定输入上需要执行多少步骤才能得到结果。

对于插入排序来说：

| 行号 | `INSERTION-SORT(A)` | 运行时间 | 运行次数 |
| :---: | :--- | :--- | :--- |
| 1 | `for j = 2 to A.length` | $c_1$ | $n$ |
| 2 | `key = A[j]` | $c_2$ | $n - 1$ |
| 3 | `i = j - 1` | $c_3$ | $n - 1$ |
| 4 | `while i > 0 and A[i] > key` | $c_4$ | $\sum_{j = 2}^n t_j$ |
| 5 | `A[i + 1] = A[i]` | $c_5$ | $\sum_{j = 2}^n (t_j - 1)$ |
| 6 | `i = i = 1` | $c_6$ | $\sum_{j = 2}^n (t_j - 1)$ |
| 7 | `A[i + 1] = key` | $c_7$ | $n - 1$ |

其中 $t_j$ 是对应 $j$ 下这一步的运行次数。最终可以得到总的运行时间：

$$
\begin{aligned}
T(n) & = c_1 n + c_2 (n - 1) + c_3 (n - 1) + c_4 \sum_{j = 2}^n t_j \\\\
& + c_5 \sum_{j = 2}^n (t_j - 1) + c_6 \sum_{j = 2}^n (t_j - 1) + c_7 (n - 1)
\end{aligned}
$$

即使是输入规模相同的实例，运行时间也会有很大差别。对于插入排序来说，最好的情况就是输入已经排好序。当代码运行到第 4 行时，循环条件永远是不满足的，$t_j = 1$。总运行时间为：

$$
\begin{aligned}
T(n) & = c_1 n + c_2 (n - 1) + c_3 (n - 1) + c_4 (n - 1) + c_7 (n - 1) \\\\
& = (c_1 + c_2 + c_3 + c_4 + c_7) n - (c_2 + c_3 + c_4 + c_7)
\end{aligned}
$$

运行时间可以被整理成 $a n + b$ 的形式，其中 $a$ 和 $b$ 是只与 $c_i$ 相关的常数。此时 $T(n)$ 称作 $n$ 的**线性函数**（linear function）。

最坏的情况发生在输入恰好是倒序时，第 4 行的循环要运行到 `i` 等于 $0$ 才停止，$t_j = j$。总运行时间为：

$$
\begin{aligned}
T(n) & =  c_1 n + c_2 (n - 1) + c_3 (n - 1) + c_4 \sum_{j = 2}^n j \\\\
& + c_5 \sum_{j = 2}^n (j - 1) + c_6 \sum_{j = 2}^n (j - 1) + c_7 (n - 1) \\\\
& = c_1 n + c_2 (n - 1) + c_3 (n - 1) + c_4 (\frac{n (n + 1)}{2} - 1) \\\\
& + c_5 \frac{n (n - 1)}{2} + c_6 \frac{n (n - 1)}{2} + c_7 (n - 1) \\\\
& = (\frac{c_4}{2} + \frac{c_5}{2} + \frac{c_6}{2}) n^2 + (c_1 + c_2 + c_3 + \frac{c_4}{2} - \frac{c_5}{2} - \frac{c_6}{2} + c_7) n \\\\
& - (c_2 + c_3 + c_4 + c_7)
\end{aligned}
$$

运行时间可以被整理成 $a n^2 + b n + c$ 的形式，其中 $a$、$b$ 和 $c$ 是只与 $c_i$ 相关的常数且 $a \ne 0$。此时 $T(n)$ 称作 $n$ 的**二次函数**（quadratic function）。

### 最坏情况分析和平均情况分析

通常来说，**最坏情况运行时间**（worst-case running time）更重要，它是输入规模为 $n$ 时的最长运行时间。最坏情况运行时间的重要性在于以下三点：

- 最坏情况运行时间是算法运行时间的上限。
- 对于某些算法来说，最坏情况经常发生。
- 平均情况运行时间通常与最坏情况运行时间大致相等。

### 增长率

**增长率**（rate of growth 或 order of growth）可以用来简化上节中得到的结果。在 $an^2 + bn + c$ 中，由于高阶的项增长更快，所以其他项通常被忽略。同时，常数也往往全被省略。因此，插入排序的最坏情况运行时间可以写作 $\Theta(n^2)$。$\Theta$ 标记将在后续章节详细介绍。

通常具有更低最坏情况运行时间增长率的算法是更优的选择。

## 2.3 设计算法

插入排序中使用了**增量**（incremental）法：保持 `A[1..j - 1]` 有序，然后把 `A[j]` 插入到适当位置，使 `A[1..j]` 依然有序。

### 分治法

许多算法使用了**递归**（recursive）结构：为了解决给定问题，算法将会调用自身，用于解决与当前问题紧密相关的子问题。这些算法通常使用了**分治**（divide-and-conquer）法：把问题拆分成数个相同类型但规模更小的子问题，递归地解决这些子问题，然后将得到的结果组合起来，构成原问题的解。

分治法的每层递归都由以下三步构成：

1. **拆分**（divide）：把问题拆分成数个相同类型但规模更小的子问题。
2. **处理**（conquer）：递归地解决这些子问题。如果子问题的规模足够小，那么就可以直接解决。
3. **组合**（combine）：将子问题的解组合起来，构成原问题的解。

以归并排序算法为例：

1. 拆分：把需要排序的 $n$ 个元素的序列拆分成两个含有 $n / 2$ 个元素的子序列。
2. 处理：使用归并排序将两个子序列排序。
3. 组合：将两个有序子序列合并到一起，得到原问题的解。

归并算法的关键步骤在于合并两个有序序列，不妨设计一个辅助函数 `MERGE(A, p, q, r)` 来完成这个步骤。其中 `A` 是数组，`p`、`q`、`r` 是数组的下标。`A[p..q]` 和 `A[q + 1..r]` 是两个有序子数组。

`MERGE` 函数花费 $\Theta(n)$ 时间，其中 $n = r - p + 1$。还是用纸牌举例，设想有两堆排好序的牌。每次迭代过程中，查看两个牌堆顶的牌，把这二者之中最小的放在手牌最后的位置。

``` plaintext
MERGE(A, p, q, r)
  // 计算子数组元素个数，并创建新数组
  n1 = q - p + 1
  n2 = r - q
  创建新数组 L[1..n1 + 1] 和 R[1..n2 + 1]

  // 将数据复制到新数组
  for i = 1 to n1
    L[i] = A[p + i - 1]
  for j = 1 to n2
    R[j] = A[q + j]

  // 在新数组最后的位置放一个无限大作为结束标志
  L[n1 + 1] = infinity
  R[n2 + 1] = infinity

  // 合并两个子数组
  i = 1
  j = 1
  for k = p to r
    // 将两个元素中较小的那个加入
    if L[i] <= R[j]
      A[k] = L[i]
      i = i + 1
    else
      A[k] = R[j]
      j = j + 1
```

最后，归并排序算法的伪代码如下：

``` plaintext
MERGE-SORT(A, p, r)
  if p < r
    // 拆分
    q = (p + r) / 2
    // 处理
    MERGE-SORT(A, p, q)
    MERGE-SORT(A, q + 1, r)
    // 组合
    MERGE(A, p, q, r)
```

归并排序的循环不变性和正确性证明略。

### 分治算法的分析

当算法递归地调用自己时，通常使用递推式来描述运行时间。假设输入实例的规模是 $n$。当 $n$ 足够小（比如 $n \leqslant c$）时可以直接解决该问题，此时花费的时间是 $\Theta(1)$（即常数时间）。当 $n > c$ 时，算法把该问题拆分成 $a$ 个规模变为原来 $1 / b$ 的子问题，一共需要花费 $a T(n / b)$ 的时间来解决这些子问题。另外，还需要 $D(n)$ 的时间来拆分问题，$C(n)$ 的时间来合并子问题的解。最终得到递推式：

$$
T(n) =
\begin{cases}
\Theta(1) & \text{如果 } n \leqslant c \\\\
a T(n / b) + D(n) + C(n) & \text{其他情况}
\end{cases}
$$

在第 4 章中将会展示如何处理这些递推式。

### 归并排序的分析

当 $n > 1$ 时，有：

- 拆分：只计算数组中间的下标，因此只花费 $D(n) = \Theta(1)$。
- 处理：递归地解决两个子问题，每个子问题的规模都是 $n / 2$，因此一共花费 $2 T(n / 2)$。
- 合并：上文中已经提到合并花费的时间是 $\Theta(n)$。

最终得到：

$$
T(n) =
\begin{cases}
\Theta(1) & \text{如果 } n = 1 \\\\
2 T(n / 2) + \Theta(n) & \text{如果 } n > 1
\end{cases}
$$

根据主定理（见第 4 章）可得 $T(n) = \Theta(n \lg n)$。不过还可以借助**递归树**（recursion tree）来解决这个问题：

``` plaintext
/                cn ---------------> cn
|              /    \
|            cn/2   cn/2 ----------> cn
|          /   \    /   \
lg n   cn/4  cn/4  cn/4  cn/4 -----> cn
|       / \   / \   / \   / \
|      ........................     ...
|      |  |  |  |  |  ... |  |
\      c  c  c  c  c      c  c ----> cn
```

在递归树的第一层中，即解决原问题的过程中，需要 $c n$ 步来合并两个子问题的解。在第二层中，也就是解决原问题的两个子问题的过程中，一共需要 $c n / 2 + c n / 2 = c n$ 步合并两个子问题的四个子子问题的解。容易发现，当递归树层数增加时每一层用于合并的步骤的总数量是不变的，总是 $c n$。另一方面，由于每次都将数组分割为两份，而且数组长度小于 $2$ 时就不再分隔，因此递归树的高度不会超过 $\lceil \lg n + 1 \rceil$。所以归并排序花费的总时间是 $T(n) = \Theta(c n \lceil \lg n + 1 \rceil) = \Theta(n \lg n)$。
