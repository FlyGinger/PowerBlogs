## 5.1 招聘问题

假设你要雇佣新员工。每次你可以花费少量资金，面试一个应聘者。该应聘者的能力是随机的，如果你决定雇佣他，你需要花费一大笔钱。只有当你决定是否雇佣此应聘者之后，你才能面试下一个应聘者。你的策略如下：如果应聘者的能力比上一个被雇佣的应聘者优秀（当没有上一个时直接雇佣），那么雇佣该应聘者；否则就不雇佣该应聘者，继续面试。这种策略的伪代码如下：

``` plaintext
HIRE-ASSISTANT(n)
  // 0 号是具有最低能力的一个虚拟应聘者
  best = 0
  for i = 1 to n
    面试应聘者 i
    if 应聘者 i 比 best 的能力优秀
      best = i
      雇佣应聘者 i
```

这次分析重点并不在运行时间，而是招聘过程中一共花了多少钱。假设面试成本是 $c_i$，雇佣成本是 $c_h$，一共面试了 $n$ 个人，最终雇佣了 $m$ 个人，那么总成本是 $O(c_i n + c_h m)$。由于 $c_i n$ 是固定的，因此 $c_h m$ 更重要。在各种程序中，遍历数组时常常需要维护最值。招聘问题实际是该类问题的建模，关注的是最值被更新的频率。

### 最坏情况分析

最差情况下，应聘者按能力从低到高来面试，这样你就花了 $O(c_h n)$ 雇佣了所有的应聘者。

当然，这种情况不大可能发生，平均情况更重要。

### 概率分析

**概率分析**（probabilistic analysis）就是在分析中使用概率。在大多数问题中，概率分析用于分析运行时间（在招聘问题中用于分析雇佣成本）。为了进行概率分析，还需要了解或者假设输入数据的分布。确定输入分布时必须万分注意。如果无法找到一个合理的输入分布，那么就不能使用概率分析。

在招聘问题中，假设所有应聘者是按随机顺序进行面试的。使用数学语言来描述，$n$ 个应聘者按面试顺序构成的序列一共有 $n!$ 种。这 $n!$ 种中的每一个出现的概率都是相同的，这称为**均匀随机排列**（uniform random permutation）。

### 随机算法

如果算法的行为不仅和输入有关，还和**随机数生成器**（random-number generator）生成的值有关，那么该算法称为**随机**（randomized）的。

要把输入的随机和算法本身的随机区分开来。对于输入具有某种概率分布的算法，研究对象是平均情况运行时间；对于本身具有随机性的算法，研究对象是**期望运行时间**（expected running time）。

## 5.2 指示随机变量

**指示随机变量**（indicator random variable）提供了概率和期望之间转换的便利方法。与事件 $A$ 关联的指示随机变量 $I\\{A\\}$ 的定义是：

$$
I\\{A\\} =
\begin{cases}
1 & \text{如果 } A \text{ 发生} \\\\
0 & \text{如果 } A \text{ 不发生} \\\\
\end{cases}
$$

以扔硬币为例，样本空间 $S = \\{H, T\\}$，且 $\Pr\\{H\\} = \Pr\\{T\\} = 1 / 2$。定义指示随机变量 $X_H$，写作：

$$
X_H = I\\{H\\} =
\begin{cases}
1 & \text{如果 } H \text{ 发生} \\\\
0 & \text{如果 } H \text{ 不发生} \\\\
\end{cases}
$$

扔一次硬币过程中，正面向上的次数的数学期望可以通过 $X_H$ 快速地得到：

$$
\begin{aligned}
E[X_H] & = E[I\\{H\\}] = 1 \cdot \Pr\\{H\\} + 0 \cdot \Pr\\{T\\} \\\\
& = 1 \cdot (1 / 2) + 0 \cdot (1 / 2) = 1 / 2
\end{aligned}
$$

### 引理 5.1

给定样本空间 $S$ 和 $S$ 中的事件 $A$，令 $X_A = I\\{A\\}$，那么 $E[X_A] = \Pr\\{A\\}$。

让问题复杂一些，令 $X_i = \\{\text{第 } i \text{ 次扔硬币时正面向上}\\}$。再假设 $X$ 表示扔 $n$ 次硬币，正面向上出现了多少次：

$$
X = \sum_{i = 1}^n X_i
$$

使用指示随机变量可以轻松地得到数学期望：

$$
\begin{aligned}
E[X_H] & = E[\sum_{i = 1}^n X_i] = \sum_{i = 1} ^ n E[X_i] \\\\
& = \sum_{i = 1}^n 1 / 2 = n / 2
\end{aligned}
$$

### 招聘问题

回到招聘问题，应聘者按随机顺序面试。令 $X$ 代表最终雇佣了多少个应聘者，希望得到数学期望：

$$
E[X] = \sum_{i = 1}^n x \Pr\\{X = x\\}
$$

这样计算起来有些困难，使用指示随机变量简化一下。定义 $X_i$：

$$
X_i = I\\{\text{第 } i \text{ 位应聘者被雇佣}\\}
\begin{cases}
1 & \text{如果第 } i \text{ 位应聘者被雇佣} \\\\
0 & \text{如果第 } i \text{ 位应聘者不被雇佣} \\\\
\end{cases}
$$

然后有：

$$
X = X_1 + X_2 + \dots + X_n
$$

根据[引理 5.1](#引理-51)，可得：

$$
E[X_i] = \Pr\\{\text{第 } i \text{ 位应聘者被雇佣}\\}
$$

此时，问题变成了求第 $i$ 位应聘者被雇佣的概率。

第 $i$ 位应聘者被雇佣当且仅当他比之前的所有应聘者都优秀。先考虑另一个问题：假设有 $x$ 个各不相同的自然数，那么这 $x$ 个自然数构成的排列有 $x!$ 种。其中，最大的数位于最后一个位置的排列有 $(x - 1)!$ 种。也就是说，在 $x$ 个自然数构成的随机排列中，最大的数恰好位于最后的概率是 $1 / x$。同理，第 $i$ 位应聘者被雇佣的概率就是 $1 / i$。于是有（用到了调和级数 $\sum_{k = 1}^n \frac{1}{k} = \ln n + O(1)$）：

$$
\begin{aligned}
E[X] & = E[\sum_{i = 1}^n X_i] = \sum_{i = 1}^n E[X_i] \\\\
& = \sum_{i = 1}^n 1 / i = \ln n + O(1)
\end{aligned}
$$

因此，当一共面试 $n$ 个应聘者时，平均会雇佣 $\ln n$ 个人。

### 引理 5.2

如果应聘者按随机顺序被面试，那么 `HIRE-ASSISTANT` 算法在平均情况下的雇佣成本是 $O(c_h \ln n)$。

## 5.3 随机算法

有时候不能对输入做出任何假设，并且输入本身也不满足任何分布，因此无法使用概率分析。这时可以强行分布输入，比如随机打乱应聘者的顺序，使其满足均匀随机排列，这样的算法就是一种随机算法。

``` plaintext
RANDOMIZED-HIRE-ASSISTANT(n)
  随机排列应聘者列表
  // 0 号是具有最低能力的一个虚拟应聘者
  best = 0
  for i = 1 to n
    面试应聘者 i
    if 应聘者 i 比 best 的能力优秀
      best = i
      雇佣应聘者 i
```

再强调一下概率分析和随机算法的不同。在 `HIRE-ASSISTANT` 的概率分析中，假定应聘者按随机顺序到来。一定要注意该算法是确定性的，对于同一个输入，算法的输出永远是相同的。而在 `RANDOMIZED-HIRE-ASSISTANT` 中，由于打乱应聘者顺序的过程是随机的，所以即使对于同一个输入，算法的输出也可能不同。

### 引理 5.3

`RANDOMIZED-HIRE-ASSISTANT` 算法的期望雇佣成本是 $O(c_h \ln n)$。

### 随机排列数组

许多随机算法都会把输入数组随机排列，这里介绍几种随机排列数组的方法。

第一种就是随机为每个位置赋予一个权重 `P[i]`，然后根据权重来排序。

``` plaintext
PERMUTE-BY-SORTING(A)
  n = A.length
  创建新数组 P[1..n]
  for i = 1 to n
     P[i] = RANDOM(1, n^3)
  根据 P 中的优先级为 A 排序
```

`RANDOM(1, n^3)` 会随机生成一个位于 `[1, n^3]` 区间内的整数。选择 `n^3` 而不是 `n` 的原因是要尽可能避免不同位置的权重相同。如果排序使用的算法是 $\Theta(n \lg n)$ 的，那么 `PERMUTE-BY-SORTING` 算法的运行时间是 $O(n \lg n)$ 的。

### 引理 5.4

如果所有权重各不相同，那么 `PERMUTE-BY-SORTING` 生成的排列满足均匀随机排列。

然后介绍一个运行时间更少的算法：

``` plaintext
RANDOMIZE-IN-PLACE(A)
  n = A.length
  for i = 1 to n
    将 A[i] 和 A[RANDOM(i, n)] 交换
```

### 引理 5.5

`RANDOMIZE-IN-PLACE` 生成的排列满足均匀随机排列。
