动态规划（ DP ， dynamic programming ）是一种类似于分治法的方法，通过组合子问题的解来得到当前问题的解。在第 2 章中介绍过的分治法把需要求解的问题分解成互斥的子问题，递归地求解这些子问题，然后组合子问题的解，最终获得当前问题的解。与分治法不同的是，动态规划一般应用于子问题重叠的场景。换句话说，子问题之间有共同的子子问题。在这种情况下，分治法会进行不必要的重复计算，重复地计算那些共同的子子问题的解。动态规划只会计算一次子子问题的解，并将其记录在表格中（此之谓 programming ），从而提升运行速度。

通常，动态规划用于解决**最优问题**（ optimization problem ）。这类问题可能有多个解，每个解有一个数值，我们希望得到最优（最大或最小）的数值。这个最优数值称为该问题的最优解。

构造一个问题的动态规划算法时，一般需要以下四步：

1. 概括最优解的结构特征。
2. 递归定义最优解的数值。
3. 计算最优解，通常使用自底向上方法。
4. 从计算出的信息中构造最优解。

步骤 1 到 3 是动态规划的基础步骤。如果我们不仅需要最优解的数值，还需要最优解本身，那么还需要执行步骤 4 。

## 15.1 棍棒切割

动态规划的第一个例子是棍棒切割问题。举个例子，某公司购买了很多长钢管，想要把它们切割成短钢管出售，切割是免费的，该公司想要知道最佳的切割方案是什么。

**棍棒切割问题**（ rod-cutting problem ）的形式化描述如下：给定一根长度为 $n$ 的长棍棒，以及一个价目表。价目表中给出了长度为 $i$ 的棍棒的价格 $p_i$ ，其中 $i = 1, 2, \dots, n$ 。请计算最大的利润 $r_n$ 。

长度为 $n$ 的棍棒一共有 $2^{n-1}$ 种切割方法，因为在每个单位长度处都有切或不切两种选择。显然，将长棍棒切成两根短棍棒时，长棍棒的利润等于两个短棍棒的利润之和。于是，有最简单直接的递归实现：

``` plaintext
CUT-ROD(p, n)
  if n == 0
    return 0
  q = - infinity
  for i = 1 to n
    q = max(q, p[i] + CUT-ROD(p, n-i))
  return q
```

在上面的算法中，对于输入的长度为 $n$ 的棍棒，每层递归都会尝试切下一部分 $i$ 并出售，最终切完后得到总利润。问题在于，递归调用的过程中对于输入相同的问题重复求解。以输入 4 为例，递归树如下：

![递归树](/notes/算法导论/images/15-00.png#center)

可以看到，对于输入是 2 、 1 和 0 的问题都求解了数次，重复求解大幅提升了运行时间。

$$
T(n) = 1 + \sum_{j=0}^{n-1} T(j)
$$

使用数学归纳法可以证明 $T(n) = 2^n$ 。

动态规划方法能够有效地提示运行速度。在解决每个子问题时，基于动态规划的算法都会将该子问题的解记录下来，以后再次需要求解该子问题时，就可以直接使用记录下来的解。动态规划使用了额外的空间，减少了运行时间，是一种以空间换时间的技巧，是时空权衡（ time-memory trade-off ）的典型案例。通过使用额外的空间，动态规划算法可以将某些指数运行时间的算法转换为多项式运行时间的算法。

有两种实现动态规划的方法，其一是**记忆化的自顶向下**（ top-down with memoization ）的方法。在这种方法中，我们还是按原来的递归写法进行编码，但是会在求解结束之后保存结果，并且会在计算之前查询是否已经求解过该问题。其二是**自底向上**（ bottom-up ）的方法，我们先求小规模的问题的解，并将其保存起来，然后去求解更大规模的问题。

> Memoization 确实没有拼写错误，它的词根是 memo 。

在实践中，往往是自底向上的方法更有效率。

``` plaintext
BOTTOM-UP-CUT-ROD(p, n)
  let r[0..n] be a new array
  r[0] = 0
  for j = 1 to n
    q = - infinity
    for i = 1 to j
      q = max(q, p[i] + r[j - i])
    r[j] = q
  return r[n]
```

上面的算法可以给出最高的利润，但是无法给出切割方案，以下算法可以打印出切割方案。

``` plaintext
EXTENDED-BOTTOM-UP-CUT-ROD(p, n)
  let r[0..n] and s[0..n] be new arrays
  r[0] = 0
  for j = 1 to n
    q = - infinity
    for i = 1 to j
      if q < p[i] + r[j - i]
        q = p[i] + r[j - i]
        s[j] = i
      r[j] = q
  return r and s

PRINT-CUT-ROD-SOLUTION(p, n)
  r, s = EXTENDED-BOTTOM-UP-CUT-ROD(p, n)
  while n > 0
    print s[n]
    n = n - s[n]
```

## 15.2 矩阵连乘

假设矩阵序列 $\lang A_1, A_2, A_3 \rang$ 代表 $A_1 A_2 A_3$ 。如果这三个矩阵的维度分别是 $10 \times 100$ 、 $100 \times 5$ 、 $5 \times 50$ 。如果先计算 $A_1 A_2$ ，然后将结果乘以 $A_3$ ，那么需要计算 $10 \times 100 \times 5 + 10 \times 5 \times 50 = 7500$ 次标量乘法。如果先计算 $A_2 A_3$ ，然后将 $A_1$ 乘以结果，那么需要计算 $100 \times 5 \times 50 + 10 \times 100 \times 50 = 75000$ 次标量乘法。显然，运算顺序对运算量有很大影响。

**矩阵连乘问题**（ matrix-chain multiplication problem ）的形式化定义如下：对于 $n$ 个矩阵 $\lang A_1, A_2, \dots, A_n \rang$ ， $A_i$ 的维度是 $p_{i-1} \times p_i$ ，其中 $i = 1, 2, \dots, n$ 。请求出计算 $A_1 A_2 \dots A_n$ 所需要的最小标量乘法数量。

可以使用括号来确定运算顺序，运算顺序完全确定的乘式是**完全括号化的**（ fully parenthesized ）。如果使用形式化的定义，那么完全括号化的乘式要么是单个的矩阵，要么是两个完全括号化的乘式相乘。举个例子， $\lang A_1, A_2, A_3, A_4 \rang$ ，一共有五种完全括号化的方式： $( A_1 ( A_2 ( A_3 A_4 ) ) )$ ， $( A_1 ( ( A_2 A_3 ) A_4 ) )$ ， $( ( A_1 A_2 ) ( A_3 A_4 ) )$ ， $( ( A_1 ( A_2 A_3 ) ) A_4 )$ ， $( ( ( A_1 A_2 ) A_3 ) A_4 )$ 。

设 $n$ 个矩阵的乘式有 $P(n)$ 种完全括号化的方式。当 $n = 1$ 时，显然只有一种方式使其完全括号化。当 $n \geqslant 2$ 时，完全括号化的乘式是两个完全括号化的子乘式的相乘。因此，递推公式：

$$
P(n) =
\begin{cases}
1 & \text{if } n = 1 \\\\
\sum_{k=1}^{n-1} P(k) P(n-k) & \text{if } n \geqslant 2
\end{cases}
$$

该递归式的解与**卡塔兰数**（ Catalan numbers ）类似，是指数速度增长的，我们难以遍历所有完全括号化的方式。

使用动态规划方法。第一步，概括最优解的结构特征。我们使用 $A_{i..j}$ 表示 $A_i A_{i+1} \dots A_{j}$ 。为了使其完全括号化，我们需要从 $A_k$ 和 $A_{k+1}$ 之间把 $A_{i..j}$ 拆分成两部分，其中 $i \leqslant k < j$ 。也就是说，首先计算 $A_{i..k}$ 和 $A_{k+1..j}$ ，最后计算二者的乘积。假设在 $A_k$ 和 $A_{k+1}$ 之间拆分能够得到最优解，那么我们在 $A_{i..j}$ 中完全括号化 $A_{i..k}$ 的方式也是 $A_{i..k}$ 的最优解。这就是所谓的最优子结构。子问题的最优解组合起来可以得到当前问题的最优解，是动态规划问题正确性的保证。这些都可以用反证法简单地证明出来。

第二步，递归定义最优解的数值。根据之前的分析，可以得到递推式：

$$
m[i,j] =
\begin{cases}
0 & \text{if } i = j \\\\
\min_{i \leqslant k < j} \\{m[i,k] + m[k+1,j] + p_{i-1} p_j \\} & \text{if } i < j
\end{cases}
$$

第三步，计算最优解。

``` plaintext
MATRIX-CHAIN-ORDER(p)
  n = p.length - 1
  let m[1..n, 1..n] and s[1..n-1, 2..n] be new tables
  for i = 1 to n
    m[i,i] = 0
  for l = 2 to n // l if the chain length
    for i = 1 to n - l + 1
      j = i + l - 1
      m[i,j] = infinity
      for k = i to j - 1
        q = m[i,k] + m[k+1,j] + p[i-1] * p[k] * p[j]
        if q < m[i,j]
          m[i,j] = q
          s[i,j] = k
  return m and s
```

第四步，构造最优解。

``` plaintext
PRINT-OPTIMAL-PARENS(s, i, j)
  if i == j
    print "A"
    print i
  else
    print "("
    PRINT-OPTIMAL-PARENS(s, i, s[i,j])
    PRINT-OPTIMAL-PARENS(s, s[i,j]+1, j)
    print ")"
```

## 15.3 动态规划基础

两个例子可能不足以让人理解如何正确使用动态规划。

### 最优子结构

使用动态规划方法的第一步是概括最优解的结构特征。所谓**最优子结构**（ optimal substructure ）就是问题的一种特征，表现在某问题的最优解中隐含了该问题的子问题的最优解。以矩阵连乘问题举例，如果 $A_{i..j}$ 在 $A_{k}$ 和 $A_{k+1}$ 之间拆分能得到最优解，那么 $A_{i,j}$ 的最优解中对于 $A_{i..k}$ 部分完全括号化的方式同时也是 $A_{i..k}$ 这个子问题的最优解。再以棍棒切割问题举例，假设有长度为 100 的棍棒，并且在最优解的第一步切割中将其切分成了长度为 40 和 60 的两段。那么在输入为 100 的棍棒切割问题的最优解中对于长度 40 那部分的切割方案，同时也是输入为 40 的棍棒切割问题的最优解。

寻找最优子结构的过程的大致流程是：

1. 你发现解决问题的第一步是一个选择，比如说，棍棒切割问题中的第一次切割（需要决定在哪儿切），矩阵连乘问题中的拆分（需要决定在哪儿把乘式拆成两部分）。做完选择题之后，会有一个或多个子问题需要解决。
2. 你假设对于给定问题，你知道怎么选择能得到最优解。
3. 给定一个选择，你需要确定接下来需要计算的子问题是什么，以及如何最好地概括结果空间的特征（换句话说，决定动态规划的“记忆化”部分到底记录什么）。有个比较好的原则：能少记就少记，无法实现时再扩展。
4. 你需要证明最优解中使用的子问题的解对于子问题来说也是最优的。这里有个“剪切粘贴”（ cut-and-paste ）的证明技巧，假设子问题的解不是最优的然后推导出一个矛盾。所谓剪切粘贴，就是把最优解中子问题的“非最优解”（假设）剪掉，然后粘贴进来一个“最优的”（假设它存在），然后证明这样做能够得到一个比最优解更好的解。这是比较典型的反证法，后面会举例子。

最优子结构在两方面随着问题变化而变化：

1. 问题的最优解中使用了多少子问题的解。
2. 决定使用哪个子问题时的选项有多少。

举个例子来说，在棍棒切割问题中，如果输入是 $n$ ，那么有 $n$ 种选择，每种选择只会使用一个子问题的解。而在矩阵连乘问题中，对于输入 $A_i A_{i+1} \dots A_j$ ，有 $j - i$ 个选择，每种选择对应两个子问题。

不严格的讲，动态规划算法的运行时间是以上两部分的乘积。在棍棒切割问题中，有 $\Theta(n)$ 个子问题，我们需要做 $n$ 次选择去检查每一个子问题，于是有了 $O(n^2)$ 的运行时间。在矩阵连乘问题中，有 $\Theta(n^2)$ 个子问题，需要 $n - 1$ 次选择，于是有 $O(n^3)$ 的运行时间。

### 子问题的独立性

还有一个重要的细节，那就是不要滥用最优子结构，接下来就要举一个不能使用最优子结构的例子。给定有向图 $G = (V, E)$ 和 $u, v \in V$ ：

- **无权最短路**（ unweighted shortest path ）：寻找从 $u$ 到 $v$ 之间的最短路（经过的边最少）。这样的路径一定是简单路径，因为包含循环的路径不可能最短。
- **无权最长简单路**（ unweighted longest simple path ）：寻找从 $u$ 到 $v$ 之间的最长简单路。

首先分析无权最短路。假设 $u \ne v$ ，排除平凡情况。此时，从 $u$ 到 $v$ 到路径一定会经过某个中间节点，记为 $w$ （ $w$ 可以是 $u$ 或 $v$ ）。现在，可以把路径 $u \stackrel{p}{\leadsto} v$ 拆分两个子路径 $u \stackrel{p_1}{\leadsto} w \stackrel{p_2}{\leadsto} v$ 。显然 $p$ 中边的数量等于 $p_1$ 和 $p_2$ 中边数量之和。如果 $p$ 是从 $u$ 到 $v$ 的最短路，那么 $p_1$ 也是从 $u$ 到 $w$ 的最短路。可以使用“剪切粘贴”的技巧证明。假设存在从 $u$ 到 $w$ 的最短路 $p_1'$ ，其中边的数量要小于 $p_1$ ，那么必然有 $u \stackrel{p_1'}{\leadsto} w \stackrel{p_2}{\leadsto} v$ 这条路中含有的边的数量比 $p$ 中更少，与 $p$ 是最优解的前提矛盾。因此，可以以图中每个节点作为 $w$ ，求解从 $u$ 到 $w$ 和从 $w$ 到 $v$ 的最短路，最终得到最优解。

现在分析无权最长简单路。不同于无权最短路，无权最长简单路并没有最优子结构的特性。当我们把路径 $u \stackrel{p}{\leadsto} v$ 拆分成两个子路径 $u \stackrel{p_1}{\leadsto} w \stackrel{p_2}{\leadsto} v$ 时， $p_1$ 一定是 $u$ 和 $w$ 之间的最长路径吗？答案是否定的，原因在于“简单路”，选择 $p_1$ 和 $p_2$ 时需要注意二者之间不能构成循环。也就是说，两个子问题之间并不是**独立的**（ independent ）。

![举例](/notes/算法导论/images/15-01.png#center)

假设 $u = q$ ， $v = t$ ，显然二者之间的最长简单路是 $q \rightarrow r \rightarrow t$ 。令 $w = r$ ， $q$ 和 $r$ 之间的最长简单路显然不是 $q \rightarrow r$ 。

### 子问题的重叠

决定一个最优问题能不能使用动态规划的另一个维度是子问题空间，它必须足够“小”。换句话说，求解时需要不断求解相同的子问题，而不能不断生成新的子问题。通常来说，子问题的数量和输入规模成多项式关系。当一个递归算法重复求解相同的子问题时，我们称这个最优问题有**重叠子问题**（ overlapping subproblems ）。对比来说，使用分治法的算法往往会生成新的子问题。

在实践中，如果每个子问题都需要被至少解决一次，那么使用自底向上的方法更有效率，因为自底向上的方法节省了递归的开支。不过，如果不是所有子问题都需要被求解，有时记忆化的自顶向下方法会更快一些。

## 15.4 最长公共子序列

**最长公共子序列问题**（ LCS problem ， Longest Common Subsequence problem ）的形式化描述如下。给定序列 $X = \lang x_1, x_2, \dots, x_m \rang$ 和 $Z = \lang z_1, z_2, \dots, z_k \rang$ ，如果存在一个严格递增的 $X$ 的下标序列 $\lang i_1, i_2, \dots i_k \rang$ ，对于所有 $j = 1, 2, \dots, k$ 都满足 $x_{i_j} = z_j$ ，那么就称 $Z$ 是 $X$ 的**子序列**（ subsequence ）。给定序列 $X$ 和 $Y$ ，如果序列 $Z$ 同时是 $X$ 和 $Y$ 的子序列，那么 $Z$ 就是 $X$ 和 $Y$ 的**公共子序列**（ common subsequence ）。在最长公共子序列问题中，给定序列 $X = \lang x_1, x_2, \dots, x_m \rang$ 和 $Y = \lang y_1, y_2, \dots, y_n$ ，需要求出最长的公共子序列 $Z$ 。

为了讨论方便，这里先定义序列的**前缀**（ prefix ）。序列 $X$ 的第 $i$ 个前缀 $X_i = \lang x_1, x_2, \dots, x_i \rang$ ，其中 $i = 1, 2, \dots, m$ 。最长公共子序列问题有以下最优子结构性质：

给定序列 $X = \lang x_1, x_2, \dots, x_m \rang$ 和 $Y = \lang y_1, y_2, \dots, y_n$ ，并且 $Z = \lang z_1, z_2, \dots, z_k \rang$ 是 $X$ 和 $Y$ 的某个最长公共子序列。

- 如果 $x_m = y_n$ ，那么 $z_k = x_m = y_n$ ，并且 $Z_{k-1}$ 是 $X_{m-1}$ 和 $Y_{n-1}$ 的一个最长公共子序列。
- 如果 $x_m \ne y_n$ 并且 $z_k \ne x_m$ ，那么 $Z$ 是 $X_{m-1}$ 和 $Y$ 的一个最长公共子序列。
- 如果 $x_m \ne y_n$ 并且 $z_k \ne y_n$ ，那么 $Z$ 是 $X$ 和 $Y_{n-1}$ 的一个最长公共子序列。

找出最优子结构是最难的一步，接下来就简单多了。假设 $c[i,j]$ 是 $X_i$ 和 $Y_j$ 的最长公共子序列的长度，那么：

$$
c[i,j] =
\begin{cases}
0 & \text{if } i = 0 \text{ or } j = 0 \\\\
c[i-1, j-1] + 1 & \text{if } i,j > 0 \text{ and } x_i = y_j \\\\
\max(c[i, j-1], c[i-1, j]) & \text{if } i,j > 0 \text{ and } x_i \ne y_j
\end{cases}
$$

递归式有了，代码就非常简单了。

``` plaintext
LCS-LENGTH(X, Y)
  m = X.length
  n = Y.length
  let b[1..m, 1..n] and c[0..m, 0..n] be new tables
  for i = 1 to m
    c[i,0] = 0
  for j = 0 to n
    c[0,j] = 0
  for i = 1 to m
    for j = 1 to n
      if X[i] == Y[j]
        c[i,j] = c[i-1,j-1] + 1
        b[i,j] = "upper-left"
      else if c[i-1,j] > c[i,j-1]
        c[i,j] = c[i-1,j]
        b[i,j] = "up"
      else
        c[i,j] = c[i,j-1]
        b[i,j] = "left"
  return c[m,n] and b

PRINT-LCS(b, X, i, j)
  if i == 0 or j == 0
    return
  if b[i,j] == "upper-left"
    PRINT-LCS(b, X, i-1, j-1)
    print X[i]
  else if b[i,j] == "up"
    PRINT-LCS(b, X, i-1, j)
  else
    PRINT-LCS(b, X, i, j-1)
```

如此一来，就可以在 $\Theta(mn)$ 的运行时间内得到最长公共子序列了。

此处还有一个常用的优化手段。观察 `LCS-LENGTH(X, Y)` 中空间的使用，建立两个表格使用了 $\Theta(mn)$ 的空间。但是在下面的循环中，只会访问表格当前单元格左侧、上侧和左上三个单元格。如果只需要返回最长公共子序列的长度，而不需要构造出最长公共子序列，那么就可以减少空间的使用，只保存上一行的信息。通过这种技巧，空间使用可以优化到 $\Theta(n)$ 。

## 15.5 最优二叉搜索树

**最优二叉搜索树**（ optimal binary search tree ）的形式化定义如下：给定键构成的序列 $K = \lang k_1, k_2, \dots, k_n \rang$ ，并且 $k_1 < k_2 < \dots < k_n$ ，我们需要构造一个由这些键构成的二叉搜索树。已知在树中搜索键 $k_i$ 的概率是 $p_i$ 。有时，使用者还会搜索不存在于 $K$ 中的键，因此我们使用 $n+1$ 个“空键” $d_0, d_1, \dots, d_n$ 来表示这些空键。其中， $d_0 < k_1$ ， $k_n < d_n$ ，并且 $1 < i < n$ 时满足 $k_i < d_i < k_{i+1}$ 。已知在树中搜索空键 $d_i$ 的概率是 $q_i$ 。显然：

$$
\sum_{i=1}^{n} p_i + \sum_{i=0}^{n} q_i = 1
$$

根据以上已知条件，我们希望构造出来的二叉搜索树的期望搜索花费最小。假定实际的搜索花费等于在树中检查的节点的数量，那么：

$$
\begin{aligned}
E[\text{search cost in } T] & = \sum_{i=1}^n (\text{depth}_T(k_i) + 1) * p_i + \sum\_{i=0}^n (\text{depth}_T(d_i) + 1) * q_i \\\\
& = 1 + \sum\_{i=1}^n \text{depth}_T(k_i) * p_i + \sum\_{i=0}^n \text{depth}_T(d_i) * q_i
\end{aligned}
$$

其中， $\text{depth}_T$ 代表节点在树 $T$ 中的深度，根节点的深度为 0 。期望搜索花费最小的二叉搜索树，称为最优二叉搜索树。

首先还是来寻找最优子结构。考虑最优二叉搜索树 $T$ 的任意子树 $T'$ ，它必定包含连续的键 $k_i, \dots, k_j$ ，其中 $1 \leqslant i \leqslant j \leqslant n$ 。同时，子树 $T'$ 还会包含空键 $d_{i-1}, \dots, d_j$ 。通过反证法可以证明，最优二叉搜索树 $T$ 的子树 $T'$ 同时也是对应 $k_i, \dots, k_j$ 这个子问题的最优解。

有了最优子结构，接下来需要递归定义最优解的数值。假设 $e[i,j]$ 是 $k_i, \dots, k_j$ 对应的最优二叉搜索树的期望搜索花费。对于 $k_i, \dots, k_j$ 来说，要搜索的键落在这个区间（包括隐含的空键）的概率是：

$$
w(i,j) = \sum_{l=i}^j p_l + \sum_{l=i-1}^j q_l
$$

假设树的根节点是 $k_r$ ，得到 $e[i,j]$ 的递归定义：

$$
e[i,j] = p_r + (e[i,r-1] + w(i,r-1)) + (e[r+1,j] + w(r+1,j))
$$

这一步是比较迷惑的，需要解释一下。当一棵树 $T$ 成为另一棵树根节点的子树时，其期望搜索花费会怎么变化呢？由于 $T$ 中所有节点的深度都增加了 1 ，显然期望搜索花费也会增加，而增加的幅度恰好等于 $T$ 中所有键被搜索的概率之和。因此也就有了 $e[i,j] + w(i,j)$ 这种形式。由于：

$$
w(i,j) = w(i,r-1) + p_r + w(r+1,j)
$$

所以：

$$
e[i,j] = e[i,r-1] + e[r+1,j] + w(i,j)
$$

最终，递归定义：

$$
e[i,j] =
\begin{cases}
q_i & \text{if } j = i - 1 \\\\
\min_{i \leqslant r \leqslant j}(e[i,r-1] + e[r+1,j] + w(i,j)) & \text{if } j \geqslant i
\end{cases}
$$

有了递归定义，算法也就出来了：

``` plaintext
OPTIMAL-BST(p, q, n)
  let e[1..n+1,0..n], w[i..n+1,0..n], and root[1..n,1..n] be new tables
  for i = 1 to n+1
    e[i,i-1] = q[i-1]
    w[i,i-1] = q[i-1]
  for l = 1 to n
    for i = 1 to n-l+1
      j = i + l - 1
      e[i,j] = infinity
      w[i,j] = w[i,j-1] + p[j] + q[j]
      for r = i to j
        t = e[i,r-1] + e[r+1,j] + w[i,j]
        if t < e[i,j]
          e[i,j] = t
          root[i,j] = r
  return e and root
```
